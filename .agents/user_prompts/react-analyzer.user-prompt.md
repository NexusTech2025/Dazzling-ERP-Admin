Analyze the following React codebase using your complete Frontend Architect framework.

Before reviewing the code:

1. Infer the business purpose of the component or feature.
2. Identify the component type (Page, Layout, Feature, Shared, UI, Hook, Context, Provider, etc.).
3. Explain its role within the application.
4. Identify any architectural assumptions.

Then perform a comprehensive review covering:

- Architecture
- Component design
- SOLID / DRY / KISS / YAGNI
- React patterns
- State management
- Hooks
- Rendering behavior
- Performance
- Memoization opportunities
- Accessibility (WCAG)
- Security
- TypeScript quality
- Code readability
- Maintainability
- Scalability
- Developer Experience (DX)

For every issue include:
- Issue
- Root Cause
- Impact
- Recommendation
- Priority
- Estimated Benefit

Conclude with:
- Overall code quality score (0–10)
- Performance score (0–10)
- Maintainability score (0–10)
- Scalability score (0–10)
- Production readiness score (0–10)

Only recommend changes that provide meaningful, measurable improvements. Avoid unnecessary rewrites and preserve existing business logic unless it is incorrect.
